Preface

Please note that xCash Klassic and its software are constantly evolving; it probably won't take long for some of the information here to become outdated. Always consult the official project channels for the latest details.

Content of the Package

You just installed the xCash GUI wallet for Windows, version {#GuiVersion}. The wallet enables you to send and receive xCash Klassic in a secure and privacy-preserving way.

This installer is focused on the GUI wallet and related command-line tools. It is configured to use remote nodes by default, so you can start using xCash Klassic without running your own full node on this machine.

For checking whether there are newer versions of this package, please visit the official X-Cash project site or release pages linked from the community channels.

Upgrading

If you already have a release of the xCash Klassic GUI wallet on your computer that was installed with the help of this installer (in an earlier version), upgrading is easy: just run the new installer; there is no need to uninstall the old release first.

If you previously downloaded a zip file and unpacked a “portable” GUI wallet into a folder, do not try to upgrade by pointing this installer at that same folder — it can make uninstalling later messy.

Instead, let this installer place the software into its own folder and then remove the old folder manually (after moving away any extra files you may have stored there). Wallet files and other user data are normally stored in separate locations and are not deleted by simply removing the old program folder.

Access to the Blockchain

Any xCash Klassic wallet needs access to the blockchain, the ongoing ledger of all xCash Klassic transactions. For the GUI wallet there is one principal ways to get that access:

  • Use a remote node (default): the wallet connects over the network to a node operated by you on another machine or by a third party.

For many users, especially on laptops or limited hardware, using a trusted remote node is a practical compromise that allows you to use xCash Klassic without storing the full chain locally.

Initial Synchronization

Even when using a remote node, your wallet must still scan the blockchain to discover transactions belonging to your addresses. This process is called synchronization or “sync”.

The time required depends on several factors, including network speed, node responsiveness, and how long it has been since the wallet was last opened. The first sync for a brand-new wallet may take longer than subsequent sessions.

Troubleshooting

The xCash Klassic software and especially the GUI wallet are still evolving. Occasionally, things can go wrong or behave in unexpected ways.

Despite technical problems you may encounter, your xCash Klassic coins are almost always safe: you may temporarily be unable to move them or see the correct balance, but you usually won’t lose funds as long as you keep your recovery information secure.

The most critical piece of information is your wallet’s seed (or equivalent backup). Never lose your seed, and never share it with anyone.

In the Utilities sub-folder of the xCash Klassic program group there are several icons that may help you solve problems. These are the icons with an x in front and a descriptive name in parentheses to make them visually stand out from the “everyday” entries.

Here is an overview of the utilities installed with this GUI package:

x (Try GUI Wallet Low Graphics Mode) Run the GUI wallet in a mode that is more tolerant of low-end or virtualized graphics hardware. Use this if the normal wallet window fails to appear, renders incorrectly, or feels very slow.
x (Check GUI Wallet Log) Open the log file containing status and error messages from the GUI wallet in Notepad. This can be useful for diagnostics; the last few lines often contain the most relevant information if something has gone wrong.
x (Check Default Wallet Folder) Open the standard wallet storage folder in Windows Explorer. This is useful if you want to back up your wallet files or inspect them directly.

If you encounter issues you cannot resolve, consider collecting the GUI log and any error messages and sharing them (redacting sensitive information) with the xCash-Labs community or support channels for further assistance.
